# Progress Dashboard Protocol

**목표**: 기능의 진행 상황을 checklist.md 기반으로 분석하여 시각적으로 표시합니다.

## Input Parsing

- `$1` (선택): 특정 기능의 진행률 확인
- `--all`: 모든 활성 기능의 진행률 요약

$1이 없고 --all도 없으면 가장 최근 수정된 TODO 디렉토리를 대상으로 합니다.

## Execution Steps

### 1. 대상 확인

```bash
# 특정 기능
ls TODO/$1/

# 또는 모든 기능
ls -d TODO/*/
```

### 2. Checklist 분석

각 기능의 `checklist.md`를 읽고 계산:

```
총 항목 수: `- [ ]` 또는 `- [x]`의 총 개수
완료 항목 수: `- [x]` 패턴의 개수
완료율: (완료 항목 / 총 항목) × 100
```

### 3. Progress 분석

`progress.md`를 읽고 추출:

- 최근 활동 시간
- 마지막 작업 내용
- 전체 타임라인 요약

### 5. 상태 아이콘 규칙

| 완료율 | 상태      | 아이콘 |
| ------ | --------- | ------ |
| 100%   | 완료      | ✅     |
| 70-99% | 거의 완료 | 🟢     |
| 30-69% | 진행 중   | 🟡     |
| 1-29%  | 초기 단계 | 🔴     |
| 0%     | 시작 전   | ⚪     |

### 6. 프로그레스 바 생성

```
50%:  █████░░░░░
80%:  ████████░░
100%: ██████████
```

### 7. 권장 다음 액션

- **0%**: `/do-task {feature}` - 작업 시작
- **1-99%**: `/do-task {feature}` - 작업 계속
- **100%**: `/do-commit {feature}` 후 `/do-deploy`

## Error Handling

TODO 디렉토리가 없는 경우:

```
📭 활성화된 기능이 없습니다.

새 기능을 시작하려면:
  /do-plan <feature-name>
```
